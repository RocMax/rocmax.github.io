程序员的自我修养

Date 2016-05-13(五) Tags 千里码 / Python / Baidu / Github

Github与百度爬虫

自从在GitHub建博客以来，访问量一直少得可怜，而且从Analytics的数据看，来访的大多是国外IP，估计是由于语言不通基本不会停留很久。出现这种情况当然最应该反省的是博客内容质量。但无意中得知由于前一阵的Great Cannon事件，Github封禁了百度的爬虫。尝试在百度搜索博客的关键词，果然一无所获。国内的现状就是这样，互联网市场无底线的竞争带来逆向淘汰，进而导致平台和渠道被流氓公司控制。虽然我支持Github的做法，但不让百度爬虫爬到的话国内用户基本不可能知道有这个博客。

锅甩完，问题还是得解决，学习了一下，现在有两条路可走：

  • 在国内的GitCafe(现已迁移至CODING)建立镜像，在推送Github更新的同时推送给镜像站。
  • 在VPS上建立镜像，利用Webhook实现博客更新时同步镜像。

目前我比较倾向于第二种方式，但我买的VPS是低配乞丐版，搭载的东西越来越多不知道能不能撑得住，有时间折腾下试试看吧。

安利完了，再说点感想：

  • 做题的时候发现Python真是越来越可爱了，关注于解决问题，不用考虑外围的很多杂事。C++和Python就像双持武器，要方便就用Python，要性能就上C++，战斗起来得心应手。
  • 作为一个半路出家的Coder，最初搞定一个问题，学会一点知识就觉得很了不起。后来发现知道的越多，周边的未知也越多。就像技能树一样，学会了一个节点，旁边就会拓展出更多节点，一个分支深入到尽头尚且不能，何况还有那么多分枝。每当这么想的时候就觉得以往浪费太多时间了，应该抓紧时间去学习。
